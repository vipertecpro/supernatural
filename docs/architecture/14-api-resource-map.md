# API v1 Resource Map

All routes retain the existing `data/error/meta` envelope and request ID. Public reads use `api-v1-public`; authenticated reads use `api-v1`; high-risk writes add named limits. Collections use cursor pagination by default (`page[after]`, bounded `page[size]`); bounded admin reference lists may use numbered pagination. Writes accept `Idempotency-Key` for progress batches, messages, votes, attempts, attendance, notifications/devices, and other retry-prone mobile actions.

Resources expose `id`, stable type, links/routes where useful, safe attributes, relationship linkage, `updated_at`, and optional `included` data through a bounded allowlist. Mobile receives compact resources, cache validators, server time, and conflict responses; it never depends on Inertia props. Public reads do not require verification; all protected writes do. Administrator access never automatically overrides takedown/spoiler controls without an explicit audited capability.

Prompt 4 implements published reads at `/universes/{universe}/franchises`, `/franchises/{franchise}`, `/universes/{universe}/works`, `/works/{work}`, `/works/{work}/seasons`, `/seasons/{season}`, `/seasons/{season}/episodes`, and `/episodes/{episode}`. Verified writes cover create/update, work translations, explicit publish/archive, and child-protected deletion. Catalog lists use bounded cursor pagination and allowlisted `type`, `status`, `franchise_id`, sort, include, and locale inputs.

Prompt 5 adds 28 verified authenticated routes below `/editorial`: revision listing/detail/items/blocks and lifecycle actions; explicit assignment/review/decision/application; revision citations; rights-assessment lists/create/detail; and spoiler-boundary lists/create/update. All filters/types are allowlisted, collections are cursor paginated, private notes are excluded, and stale Catalog writes use `optimistic_lock_conflict` with HTTP 409.

Prompt 6 adds public bounded routes for universe Lore, entity detail/aliases/appearances/relationships/timeline entries, universe timelines, timeline detail, and timeline entries. Verified routes create/update/publish/archive Lore roots and children, including localized text. Graph reads are one hop, cursor-bounded to 50, relationship nesting is non-recursive, and spoiler-hidden records are removed before Resource output.

Prompt 7 adds public bounded `/search`, `/search/suggestions`, `/discovery/related/{type}/{id}`, published Media detail, and allowlisted target-attachment reads. Verified Media routes create/update/publish/archive hosted assets and external embeds and create/publish/delete attachments. Storage keys, original private paths, provider internals, and legal notes never enter Resources. Search administration remains command/permission controlled rather than a public HTTP endpoint.
