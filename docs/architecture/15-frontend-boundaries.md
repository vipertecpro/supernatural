# Frontend Application Boundaries

| Surface        | Routes/layout                                                                         | Data and loading                                                                                                                                                 | Real time / performance                                                                                                    |
| -------------- | ------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| Public website | `/`, `/universes/*`, `/works/*`, `/lore/*`, `/events/*`; `PublicLayout`               | SEO-complete server/Inertia initial props, canonical metadata, structured data; deferred noncritical related content with skeleton/rescue; safe public Resources | cinematic modules dynamically imported per page; reduced-motion/no-WebGL/low-power fallbacks; no dashboard bundle coupling |
| Fan dashboard  | `/dashboard`, `/journey`, `/community`, `/messages`, `/rooms`, `/boards`; `FanLayout` | verified user, permissions, preferences; cursor lists, optimistic reversible interactions, explicit empty/error states                                           | subscribe only on relevant mounted pages; fast navigation and modest animation                                             |
| Administration | `/administration/*`, `/moderation/*`; `AdministrationLayout` and `ModerationLayout`   | server policy on every route/action; dense paginated tables, lock versions, audit/review context; no public cinematic components                                 | scoped case/review updates only; accessible predictable UI over decoration                                                 |

Shared components are low-level accessible UI primitives, error boundaries, pagination, spoiler warning, citation display, media rendering, and typed domain summaries. Public hero/WebGL/video, admin rights editors, moderation evidence viewers, and private message composers are not shared across surfaces merely for visual reuse because their data, performance, and security contracts differ.

Wayfinder generates route/action functions; Page props and API Resources have explicit TypeScript interfaces co-located by domain. Inertia shared props stay small (identity, effective capabilities, flash, request ID). Large lists use deferred props with animated skeleton and rescue/retry; sensitive props are never deferred to unauthorized users. Standalone API calls use Inertia v3 `useHttp` or the project client, not unapproved Axios.

All surfaces require keyboard navigation, visible focus, semantic landmarks, contrast, screen-reader labels, reduced motion, caption/transcript handling for owned media, alt text, zoom/reflow, and non-color status cues. Authorization is server-side. Error handling preserves request ID and provides safe recovery without exposing internals.
